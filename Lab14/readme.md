# AI Capex Diagnostic Modeling

## Objective
Built and audited an OLS revenue model for Nvidia’s 2026 AI capital expenditure environment, with a focus on detecting structural violations in the error process and correcting inference using HC3 robust standard errors.

## Methodology
- Loaded and examined a 2026 Nvidia AI capital expenditure and deployment dataset centered on **AI Software Revenue** and its operational drivers.
- Estimated a baseline **Ordinary Least Squares (OLS)** model using key infrastructure variables, including capital expenditure, data center power intensity, and cloud GPU deployment scale.
- Reviewed the naive model’s fit statistics, coefficient estimates, and p-values to establish the initial appearance of strong explanatory performance.
- Conducted **residual forensics** by plotting residuals against fitted values to visually inspect whether the error variance remained stable across the prediction range.
- Applied the **White test** to formally evaluate heteroscedasticity using the exact design matrix generated by the model specification.
- Calculated **Variance Inflation Factors (VIFs)** for each explanatory variable to diagnose multicollinearity and assess whether overlapping predictors were inflating coefficient variance.
- Re-estimated the identical regression using **HC3 heteroscedasticity-consistent robust standard errors**, preserving the coefficient estimates while correcting the uncertainty measures used for inference.
- Compared the naive and robust outputs to determine which apparent signals reflected genuine informational content and which were artifacts of violated OLS assumptions.

## Key Findings
This project showed that a model can appear statistically “clean” while still failing core structural assumptions in economically important ways. The baseline OLS regression produced strong fit statistics and extremely small p-values, creating an initial impression of high-confidence inference. However, residual diagnostics revealed a pronounced expansion in error variance at higher capital expenditure levels, indicating substantial **heteroscedasticity** in the revenue-generating process.

Once this failure was addressed using **HC3 robust standard errors**, the model’s false precision was materially reduced. Standard errors widened, and some variables that appeared highly significant under naive OLS became meaningfully less certain under robust inference. In other words, the correction did not change the economic relationships themselves, but it did change how confidently those relationships could be interpreted. From a tech-economics perspective, this is critical: in high-growth AI infrastructure markets, uncorrected models can overstate predictive certainty and encourage overly aggressive capital allocation decisions based on statistically fragile signals.
